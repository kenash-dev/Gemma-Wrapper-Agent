1. Overview & Architecture

Gemma Agent turns a locally running Google Gemma model into an autonomous AI agent that can reason, plan, read/write files, execute commands, search the web, and run Python code — all from your terminal.

ReAct Reasoning Loop

Think → Act → Observe → Repeat. The agent reasons step-by-step and uses tools to complete complex tasks.

10 Built-in Tools

File I/O, shell commands, Python execution, web search, URL fetch, grep, and semantic search.

Memory Management

Auto-compresses conversation history to fit Gemma's context window. Working memory scratchpad included.

Safety Sandbox

Blocks dangerous commands, restricts paths to your workspace, confirms destructive actions.

Project Structure

gemma-agent/ ├── main.py # CLI entry (REPL + single-task mode) ├── agent_loop.py # Core ReAct loop ├── llm_client.py # OpenAI-compatible client for Gemma ├── tool_registry.py # Tool registration & dispatch ├── memory.py # Conversation history + compression ├── planner.py # Task decomposition ├── sandbox.py # Safety: command & path validation ├── config.py # Settings via environment variables ├── requirements.txt ├── tools/ │ ├── file_ops.py # read, write, edit, list, search, grep │ ├── shell.py # Shell command execution │ ├── code_exec.py # Python code execution │ ├── web_search.py # DuckDuckGo + URL fetch │ └── knowledge.py # RAG (optional, needs extra deps) └── prompts/ ├── system.txt # Agent system prompt ├── planner.txt # Planning prompt └── tool_format.txt # Tool format reference

3. Running Gemma in Docker

You need Gemma served behind an OpenAI-compatible API. Here are the three most common approaches:

Option A: Ollama (Recommended — Easiest)

# Pull and run Ollama
docker run -d --name ollama -p 11434:11434 \
    -v ollama_data:/root/.ollama \
    ollama/ollama

# Pull the Gemma model
docker exec ollama ollama pull gemma2:27b

# Verify it's running
curl http://localhost:11434/v1/models

API endpoint: http://localhost:11434/v1
Model name: gemma2:27b (or gemma2:9b for smaller setups)

Option B: vLLM (Best Performance with GPU)

docker run -d --name gemma-vllm --gpus all \
    -p 8000:8000 \
    vllm/vllm-openai:latest \
    --model google/gemma-2-27b-it \
    --max-model-len 8192

API endpoint: http://localhost:8000/v1
Model name: google/gemma-2-27b-it

Option C: LocalAI (CPU-Optimized)

docker run -d --name localai -p 8080:8080 \
    -v localai_models:/models \
    localai/localai:latest

# Download Gemma via LocalAI's model gallery
curl http://localhost:8080/models/apply \
    -H "Content-Type: application/json" \
    -d '{"url": "github:mudler/LocalAI/gallery/gemma-2-27b-it.yaml"}'

API endpoint: http://localhost:8080/v1

Verify the Endpoint

# Test that your model responds
curl http://localhost:11434/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{
        "model": "gemma2:27b",
        "messages": [{"role": "user", "content": "Say hello"}],
        "max_tokens": 50
    }'

If you get a JSON response with a choices array, the endpoint is ready.

4. Installing the Agent

Navigate to the agent directory: 
cd path\to\gemma-agent
Install Python dependencies: 
pip install -r requirements.txt

This installs: openai, rich, duckduckgo-search, beautifulsoup4, requests

(Optional) Install RAG dependencies: 
pip install chromadb sentence-transformers

Then uncomment import tools.knowledge in main.py to enable semantic search.

Verify the install: 
python -c "from main import *; print('OK')"

5. Configuration

All settings are controlled via environment variables with sensible defaults:

VariableDefaultDescription
GEMMA_BASE_URLhttp://localhost:11434/v1Gemma API endpoint
GEMMA_MODELgemma2:27bModel name
AGENT_MAX_ITERATIONS15Max reasoning steps per task
AGENT_WORKSPACE. (current dir)Restrict file access to this directory
AGENT_CONFIRM_WRITEStrueAsk before writing/deleting files
AGENT_TEMPERATURE0.1LLM temperature (lower = more deterministic)
AGENT_MAX_TOKENS2048Max output tokens per LLM call
AGENT_CONTEXT_BUDGET6000Token budget for conversation history

Setting Variables

Windows (PowerShell):

$env:GEMMA_BASE_URL = "http://localhost:11434/v1"
$env:GEMMA_MODEL = "gemma2:9b"
python main.py

Linux / macOS (bash):

export GEMMA_BASE_URL="http://localhost:11434/v1"
export GEMMA_MODEL="gemma2:9b"
python main.py

Or use CLI flags (override env vars):

python main.py --base-url http://localhost:8000/v1 --model gemma2

8. How the Agent Works (ReAct Loop)

The agent follows the ReAct (Reason + Act) pattern. Each turn of the loop:

User: "Add error handling to server.py" │ ▼ ┌─ THOUGHT ──────────────────────────────┐ │ I need to read server.py first to │ │ understand the current code. │ └────────────────────────────────────────┘ │ ▼ ┌─ ACTION ───────────────────────────────┐ │ read_file({"path": "server.py"}) │ └────────────────────────────────────────┘ │ ▼ ┌─ OBSERVATION ──────────────────────────┐ │ (contents of server.py) │ └────────────────────────────────────────┘ │ ▼ ┌─ THOUGHT ──────────────────────────────┐ │ I see three endpoints without try/ │ │ except blocks. I'll add error handling │ │ to each one. │ └────────────────────────────────────────┘ │ ▼ ┌─ ACTION ───────────────────────────────┐ │ edit_file({"path": "server.py", ...}) │ └────────────────────────────────────────┘ │ ▼ ... (continues until done) ... │ ▼ ┌─ ANSWER ───────────────────────────────┐ │ I've added try/except blocks to all │ │ three endpoints with proper logging. │ └────────────────────────────────────────┘

The loop runs up to 15 iterations (configurable). If the model produces malformed output, it gets one retry with a format hint.

9. REPL Commands

In interactive mode, use these slash commands:

CommandDescription

/clearReset conversation history and working memory

Tip: Press Ctrl+C to interrupt a running agent action without killing the session. The agent will stop the current loop and wait for new input.

11. Safety & Sandboxing

Command Blocking

The following command patterns are automatically blocked and will never execute:

PatternReason
rm -rf /Recursive root deletion
sudoPrivilege escalation
mkfs, format, ddDisk formatting / raw writes
shutdown, rebootSystem shutdown
curl|bash, wget|bashPipe-to-shell attacks
chmod 777Overly permissive permissions
passwdPassword changes
Fork bomb patternsSystem resource exhaustion

Confirmation Prompts

These actions trigger a Y/N confirmation before executing:

  • rm, del — file/directory deletion
  • git push, git reset --hard — destructive Git operations
  • pip install, npm install — package installation
  • DROP TABLE, TRUNCATE — database destructive ops
  • All file writes via write_file and edit_file
Warning: Using --no-confirm disables all confirmation prompts. Only use this for trusted, well-tested tasks in isolated environments.

Path Restriction

The agent can only access files within the configured workspace directory. Any attempt to read, write, or navigate outside this directory is blocked.

# Restrict to a specific project
python main.py --workspace /home/user/projects/myapp

12. Troubleshooting

Connection Refused / Cannot Connect

# Check if the Docker container is running
docker ps

# Check if the API responds
curl http://localhost:11434/v1/models

# Check Docker logs
docker logs ollama

Model Not Found

# List available models in Ollama
docker exec ollama ollama list

# Pull the model if missing
docker exec ollama ollama pull gemma2:27b

Slow Responses

  • Use a smaller model: --model gemma2:9b
  • Ensure GPU passthrough is working: docker run --gpus all ...
  • Check GPU memory: nvidia-smi
  • Reduce AGENT_MAX_TOKENS to 1024

Agent Produces Malformed Output

  • This is normal for smaller models — the agent auto-retries once with a format hint
  • If persistent, try a larger model (27b is better than 9b at following formats)
  • Lower the temperature: AGENT_TEMPERATURE=0.05

Out of Memory

  • Switch to gemma2:9b (needs ~8 GB RAM / 6 GB VRAM)
  • Reduce context budget: AGENT_CONTEXT_BUDGET=3000
  • Use /clear to reset conversation history

Import Errors

# Reinstall all deps
pip install -r requirements.txt --force-reinstall

# Verify
python -c "from main import *; print('OK')"

13. Advanced: Adding Custom Tools

You can add your own tools by creating a new file in the tools/ directory:

# tools/my_tool.py
from tool_registry import tool

@tool(
    name="my_custom_tool",
    description="Does something useful",
    parameters={
        "input_text": {
            "type": "string",
            "description": "The input to process"
        }
    },
)
def my_custom_tool(input_text: str) -> str:
    # Your logic here
    result = input_text.upper()
    return f"Processed: {result}"

Then register it in main.py:

import tools.my_tool  # noqa: F401

The agent will automatically see it in the system prompt and can call it during reasoning.

Tips for Best Results

  • Be specific — "Add input validation to the /login endpoint in server.py" works better than "fix security"
  • Break large tasks — Ask for one thing at a time; the agent handles multi-step internally
  • Use /clear — Reset between unrelated tasks to avoid context pollution
  • Workspace flag — Always use --workspace to scope the agent's file access
  • 27B model — Use the 27B model for agent tasks; 9B struggles with the ReAct format

15. Desktop UI Application

In addition to the command-line REPL, Gemma Agent includes a full graphical desktop application built with Python's Tkinter. It provides the same agent capabilities with a modern dark-themed interface.

Launching the Desktop App

# Basic launch
python desktop_app.py

# With workspace restriction
python desktop_app.py --workspace C:\Projects\my-app

# With custom endpoint
python desktop_app.py --base-url http://localhost:8000/v1 --model gemma2:27b
No extra dependencies required. The desktop UI uses Python's built-in tkinter, which ships with every standard Windows Python installation.

Confirmation Dialogs

When Confirm destructive actions is enabled (default), the agent will show a Yes/No popup before:

  • Writing or editing any file
  • Running commands that match destructive patterns (rm, git push, pip install, etc.)

Click Yes to allow the action, or No to cancel it. The agent will acknowledge the cancellation and may try an alternative approach.

Stop & Clear

  • ■ Stop — Immediately interrupts the running agent loop. The agent stops after the current step. Useful if it's heading in the wrong direction.
  • 🗑 Clear — Resets the entire conversation history and working memory. Use between unrelated tasks.
